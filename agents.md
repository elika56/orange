# agents.md — Convention Reference

## 0. Purpose

This file is the authoritative convention reference for the Orange NestJS API project. Every module — whether written by a human or generated by an AI agent — must conform to the patterns defined here. When this file conflicts with any other source, this file wins. Update this file before deviating from it.

---

## 1. Project Layout & Module Structure

Per-feature folder: `src/<feature>/` (plural noun, e.g. `src/users/`, `src/orders/`).

**Required files per module:**

```
src/<feature>/
  <feature>.module.ts
  <feature>.controller.ts
  <feature>.service.ts
  dto/
    create-<entity>.dto.ts
    update-<entity>.dto.ts
    query-<entity>.dto.ts
  entities/
    <entity>.entity.ts
  <feature>.controller.spec.ts
  <feature>.service.spec.ts

test/
  <feature>.e2e-spec.ts
```

**Module wiring rules:**

- The service must appear in both `providers` and `exports` of the feature module.
- The feature module must be imported in `AppModule`.
- `AppModule` stays thin: no providers, no controllers of its own — only imports.

---

## 2. Controller Patterns

Global prefix `api` is set in `main.ts`. Controllers declare only the feature path (e.g. `@Controller('foos')`).

**Route order matters:** `GET /query` must be declared before `GET /:id` to prevent Express from matching the literal string `query` as an id parameter.

**Full CRUD + query skeleton:**

```typescript
import {
  Controller,
  Get,
  Post,
  Patch,
  Delete,
  Param,
  Body,
  Query,
  HttpCode,
  HttpStatus,
} from '@nestjs/common';
import { FoosService } from './foos.service';
import { CreateFooDto } from './dto/create-foo.dto';
import { UpdateFooDto } from './dto/update-foo.dto';
import { QueryFooDto } from './dto/query-foo.dto';

@Controller('foos')
export class FoosController {
  constructor(private readonly foosService: FoosService) {}

  // GET api/foos
  @Get()
  findAll() {
    return this.foosService.findAll();
  }

  // GET api/foos/query  — MUST come before /:id
  @Get('query')
  findByQuery(@Query() dto: QueryFooDto) {
    return this.foosService.findByQuery(dto);
  }

  // GET api/foos/:id
  @Get(':id')
  findOne(@Param('id') id: string) {
    return this.foosService.findOne(id);
  }

  // POST api/foos  — NestJS defaults POST to 201; be explicit anyway
  @Post()
  @HttpCode(HttpStatus.CREATED)
  create(@Body() dto: CreateFooDto) {
    return this.foosService.create(dto);
  }

  // PATCH api/foos/:id
  @Patch(':id')
  update(@Param('id') id: string, @Body() dto: UpdateFooDto) {
    return this.foosService.update(id, dto);
  }

  // DELETE api/foos/:id  — NestJS defaults DELETE to 200; override required
  @Delete(':id')
  @HttpCode(HttpStatus.NO_CONTENT)
  remove(@Param('id') id: string) {
    return this.foosService.remove(id);
  }
}
```

**Rules:**

- Use typed decorators only: `@Param`, `@Body`, `@Query`. Never inject raw `req` or `res`.
- Controllers are thin: zero business logic. Every non-trivial operation belongs in the service.

## 3. Service Patterns

**Method set (1:1 with controller routes):**

- `findAll(): Promise<Foo[]>`
- `findByQuery(query: QueryFooDto): Promise<QueryResult<Foo>>`
- `findOne(id: string): Promise<Foo>`
- `create(dto: CreateFooDto): Promise<Foo>`
- `update(id: string, dto: UpdateFooDto): Promise<Foo>`
- `remove(id: string): Promise<void>`

Return typed entities or `Promise<Entity>`. Never return `any`.

**Exception policy:** HTTP exceptions are thrown from the service, not the controller.

- `NotFoundException` when an entity with the given id does not exist.
- `ConflictException` for uniqueness violations.

**`findOneOrThrow` helper:** shared internally by `findOne`, `update`, and `remove` to keep the not-found guard DRY.

**Interim in-memory store:** until an ORM/repository is wired in, use a private array. The `findOneOrThrow` helper and the public method signatures are the repository seam — swapping the array for a TypeORM/Prisma repository requires changing only the service internals.

**Service skeleton:**

```typescript
import {
  Injectable,
  NotFoundException,
  ConflictException,
} from '@nestjs/common';
import { CreateFooDto } from './dto/create-foo.dto';
import { UpdateFooDto } from './dto/update-foo.dto';
import { QueryFooDto } from './dto/query-foo.dto';
import { Foo } from './entities/foo.entity';
import { QueryResult } from '../common/interfaces/query-result.interface';

@Injectable()
export class FoosService {
  // Interim in-memory store — replace with repository injection when ORM is added
  private readonly foos: Foo[] = [];
  private nextId = 1;

  findAll(): Promise<Foo[]> {
    return Promise.resolve([...this.foos]);
  }

  findByQuery(query: QueryFooDto): Promise<QueryResult<Foo>> {
    const { page = 1, limit = 20 } = query;
    const data = this.foos.slice((page - 1) * limit, page * limit);
    return Promise.resolve({ data, total: this.foos.length, page, limit });
  }

  async findOne(id: string): Promise<Foo> {
    return this.findOneOrThrow(id);
  }

  async create(dto: CreateFooDto): Promise<Foo> {
    const foo: Foo = { id: String(this.nextId++), ...dto };
    this.foos.push(foo);
    return foo;
  }

  async update(id: string, dto: UpdateFooDto): Promise<Foo> {
    const foo = await this.findOneOrThrow(id);
    Object.assign(foo, dto);
    return foo;
  }

  async remove(id: string): Promise<void> {
    const index = this.foos.findIndex((f) => f.id === id);
    if (index === -1) throw new NotFoundException(`Foo #${id} not found`);
    this.foos.splice(index, 1);
  }

  // Shared guard — keeps not-found logic in one place
  private async findOneOrThrow(id: string): Promise<Foo> {
    const foo = this.foos.find((f) => f.id === id);
    if (!foo) throw new NotFoundException(`Foo #${id} not found`);
    return foo;
  }
}
```

## 4. DTO & Validation Patterns

**Prerequisites:**

```bash
npm install class-validator class-transformer @nestjs/mapped-types
```

Add a global `ValidationPipe` in `src/main.ts`:

```typescript
app.useGlobalPipes(
  new ValidationPipe({ whitelist: true, transform: true }),
);
```

**`CreateDto`** — decorated fields with `class-validator`:

```typescript
import { IsString, IsNotEmpty, IsOptional } from 'class-validator';

export class CreateFooDto {
  @IsString()
  @IsNotEmpty()
  name: string;

  @IsString()
  @IsOptional()
  description?: string;
}
```

**`UpdateDto`** — `PartialType` eliminates duplication:

```typescript
import { PartialType } from '@nestjs/mapped-types';
import { CreateFooDto } from './create-foo.dto';

export class UpdateFooDto extends PartialType(CreateFooDto) {}
```

**`QueryDto`** — optional filters + pagination with numeric coercion:

```typescript
import { IsOptional, IsString, IsInt, Min } from 'class-validator';
import { Type } from 'class-transformer';

export class QueryFooDto {
  @IsString()
  @IsOptional()
  name?: string;

  @Type(() => Number)
  @IsInt()
  @Min(1)
  @IsOptional()
  page?: number = 1;

  @Type(() => Number)
  @IsInt()
  @Min(1)
  @IsOptional()
  limit?: number = 20;
}
```

**Standardized query response envelope** — every `findByQuery` route must return this shape:

```typescript
export interface QueryResult<T> {
  data: T[];
  total: number;
  page: number;
  limit: number;
}
```

## 5. Unit Test Patterns

**Prerequisites:**

```bash
npm install --save-dev jest ts-jest @types/jest @nestjs/testing
```

Add to `package.json`:

```json
{
  "scripts": {
    "test": "jest",
    "test:watch": "jest --watch",
    "test:cov": "jest --coverage"
  },
  "jest": {
    "moduleFileExtensions": ["js", "json", "ts"],
    "rootDir": "src",
    "testRegex": ".*\\.spec\\.ts$",
    "transform": { "^.+\\.(t|j)s$": "ts-jest" },
    "collectCoverageFrom": ["**/*.(t|j)s"],
    "coverageDirectory": "../coverage",
    "testEnvironment": "node"
  }
}
```

**Controller spec — mock the service, assert delegation:**

```typescript
import { Test, TestingModule } from '@nestjs/testing';
import { FoosController } from './foos.controller';
import { FoosService } from './foos.service';

describe('FoosController', () => {
  let controller: FoosController;
  let service: jest.Mocked<FoosService>;

  beforeEach(async () => {
    const module: TestingModule = await Test.createTestingModule({
      controllers: [FoosController],
      providers: [
        {
          provide: FoosService,
          useValue: {
            findAll: jest.fn(),
            findByQuery: jest.fn(),
            findOne: jest.fn(),
            create: jest.fn(),
            update: jest.fn(),
            remove: jest.fn(),
          },
        },
      ],
    }).compile();

    controller = module.get(FoosController);
    service = module.get(FoosService);
  });

  describe('findAll', () => {
    it('delegates to service and returns result', async () => {
      const foos = [{ id: '1', name: 'a' }];
      service.findAll.mockResolvedValue(foos as any);
      expect(await controller.findAll()).toBe(foos);
      expect(service.findAll).toHaveBeenCalledTimes(1);
    });
  });

  // One describe per method following the same AAA pattern
});
```

**Service spec — instantiate with clean state, test happy path + error branches:**

```typescript
import { NotFoundException } from '@nestjs/common';
import { FoosService } from './foos.service';
import { CreateFooDto } from './dto/create-foo.dto';

describe('FoosService', () => {
  let service: FoosService;

  beforeEach(() => {
    service = new FoosService();
  });

  describe('findOne', () => {
    it('returns the foo when it exists', async () => {
      const created = await service.create({ name: 'test' } as CreateFooDto);
      const found = await service.findOne(created.id);
      expect(found).toEqual(created);
    });

    it('throws NotFoundException when id is unknown', async () => {
      await expect(service.findOne('999')).rejects.toThrow(NotFoundException);
    });
  });

  // One describe per method
});
```

**Structure rules:**

- One `describe` block per method.
- AAA (Arrange / Act / Assert) within each `it`.
- `beforeEach` resets mocks or reinstantiates the service with clean state.

## 6. E2E Test Patterns

**Prerequisites:**

```bash
npm install --save-dev supertest @types/supertest
```

Create `test/jest-e2e.json`:

```json
{
  "moduleFileExtensions": ["js", "json", "ts"],
  "rootDir": ".",
  "testEnvironment": "node",
  "testRegex": ".e2e-spec.ts$",
  "transform": { "^.+\\.(t|j)s$": "ts-jest" }
}
```

Add to `package.json` scripts:

```json
"test:e2e": "jest --config ./test/jest-e2e.json"
```

**E2E bootstrap** — mirror `main.ts` exactly (same pipe, same prefix):

```typescript
import { Test, TestingModule } from '@nestjs/testing';
import { INestApplication, ValidationPipe } from '@nestjs/common';
import * as request from 'supertest';
import { AppModule } from '../src/app.module';

describe('Foos (e2e)', () => {
  let app: INestApplication;

  beforeEach(async () => {
    const moduleFixture: TestingModule = await Test.createTestingModule({
      imports: [AppModule],
    }).compile();

    app = moduleFixture.createNestApplication();
    app.setGlobalPrefix('api');
    app.useGlobalPipes(new ValidationPipe({ whitelist: true, transform: true }));
    await app.init();
  });

  afterAll(async () => {
    await app.close();
  });

  it('POST /api/foos → 201', () =>
    request(app.getHttpServer())
      .post('/api/foos')
      .send({ name: 'test' })
      .expect(201));

  it('GET /api/foos → 200', () =>
    request(app.getHttpServer()).get('/api/foos').expect(200));

  it('GET /api/foos/:id → 404 on unknown id', () =>
    request(app.getHttpServer()).get('/api/foos/999').expect(404));

  it('POST /api/foos → 400 on invalid body', () =>
    request(app.getHttpServer()).post('/api/foos').send({}).expect(400));

  it('GET /api/foos/query → 200 with envelope', () =>
    request(app.getHttpServer())
      .get('/api/foos/query?page=1&limit=5')
      .expect(200)
      .expect((res) => {
        expect(res.body).toHaveProperty('data');
        expect(res.body).toHaveProperty('total');
        expect(res.body).toHaveProperty('page');
        expect(res.body).toHaveProperty('limit');
      }));
});
```

**Coverage requirements per module:** each CRUD happy path, 404 on unknown id, 400 on invalid body, query route with pagination envelope check.

Reset in-memory state between tests by relying on `beforeEach` to rebuild the application (which reinstantiates the service with an empty array). If shared state persists, expose a `reset()` method on the service or use `moduleFixture.close()` in `afterEach`.

## 7. Code Style Rules

**DRY:**
- Use `PartialType(CreateDto)` — never copy-paste fields into `UpdateDto`.
- Use `findOneOrThrow` — never inline the not-found guard in multiple methods.
- Export services so other modules can inject them without re-implementing logic.

**Naming:**
- Feature folders and URL paths: `plural-kebab-case` (`src/foo-bars/`, `@Controller('foo-bars')`).
- Classes: `PascalCase`.
- Filenames: `kebab-case`.
- DTOs: `CreateFooDto`, `UpdateFooDto`, `QueryFooDto` — always the `Dto` suffix.
- Entities: `Foo`, defined in `entities/foo.entity.ts` — always the `Entity` suffix on the filename.

**Strict typing:**
- `strict: true` is on. No `any`. All functions have explicit return types.
- Lean on `Promise<T>` even for synchronous stubs so swapping to async persistence is non-breaking.

**HTTP verbs:**
- Use `PATCH`, not `PUT`, for partial updates.

**Comments:**
- Explain WHY, never WHAT. If the comment restates the code, delete it.
- No commented-out code in committed files.

## 8. New-Module Checklist

Follow this checklist in order when adding any new module:

1. Create `src/<feature>/` with all 7 source files: module, controller, service, 3 DTOs, 1 entity.
2. Implement the full route set in the controller: `findAll`, `findByQuery` (before `/:id`), `findOne`, `create` (201), `update`, `remove` (204).
3. Implement the service with all 6 public methods, `findOneOrThrow` helper, `NotFoundException` on missing entity, `ConflictException` for duplicates.
4. Create `CreateDto` (validated fields), `UpdateDto` (`PartialType(CreateDto)`), `QueryDto` (optional filters + `page`/`limit` with `@Type(() => Number)`).
5. Create the entity interface or class in `entities/<entity>.entity.ts`.
6. Write the controller unit spec: `Test.createTestingModule`, service fully mocked with `jest.fn()`, one `describe` per method.
7. Write the service unit spec: instantiate with `new`, cover happy path and every error branch, one `describe` per method.
8. Write the e2e spec in `test/<feature>.e2e-spec.ts`: full `AppModule` bootstrap, same pipe + prefix as `main.ts`, cover all routes + 404 + 400 + query envelope.
9. Import the feature module in `AppModule`.
10. Run `npm test` and `npm run test:e2e` — all green before committing.
