# Project conventions

## Conventions reference

Always read [agents.md](agents.md) before writing or modifying any code. It is the authoritative convention reference for this NestJS project — module structure, naming, patterns, and more.

## Pull requests

When creating PR descriptions (via `/feature-pr` or any `gh pr create`), always use emojis to make the description visually appealing. Use the following structure:

- `## ✨ Summary` — bullet points with relevant emojis (🗂️ for files/entities, 🔌 for wiring/controllers, 🛠️ for config, etc.)
- `## 🧪 Test plan` — checklist items prefixed with ✅
- Footer: `🤖 Generated with [Claude Code](https://claude.ai/claude-code)`
